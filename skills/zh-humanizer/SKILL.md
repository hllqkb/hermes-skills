# 中文 AI 文章润色师

## When to Use

- 用户说"去 AI 味"、"润色"、"改成人话"、"humanize"、"润色一下"
- 用户觉得文章太机械、太模板化、读起来像 AI 写的
- 用户有一篇 AI 生成的中文文章需要改得更自然

## 默认参数（用户可覆盖）

- 语言：中文
- 目标读者：普通大众
- 语调：幽默风趣、自然流畅
- 如果用户指定了其他受众或风格，按用户要求来

## Workflow

### Step 1: 原文诊断

快速阅读原文，识别 2-3 个最突出的"AI 味"问题：

- **句式单一**：大量"XXX 是 XXX"、"XXX 具有 XXX"的判断句，缺少变化
- **过渡生硬**：满篇"首先/其次/此外/总之"、"值得注意的是"、"不难发现"、"基于以上分析"
- **情感缺失**：全是客观陈述，没有个人视角、没有感受、没有"人味"
- **列表依赖**：动不动就用表格和 bullet point，明明可以写成流畅的段落
- **套话堆砌**：大量"核心理念"、"关键在于"、"本质上"、"综上所述"等空洞词汇
- **排比滥用**：三个一组的排比句密集出现，像在写高考作文

### Step 2: 核心优化策略

根据诊断结果，选择 3-5 个优化方向：

1. **句式灵动**：长短句结合，引入倒装、设问、感叹。告别"XXX 是 XXX"的判断句轰炸
2. **词汇鲜活**：用具体、有温度的词替换抽象、生硬的词。多用动词，少用被动
3. **自然过渡**：抛弃"首先/其次/总之"，用"说到这里"、"回过头来看"、"有意思的是"、"更关键的是"等隐性连接
4. **情感注入**：适度引入第一人称、感叹、反问。展示而非说教（Show, don't tell）
5. **互动感**：用设问拉近距离，如"你可能会想……"、"说实话，这一步我自己也踩过坑"
6. **去表格化**：能用段落说清楚的，不要硬拆成表格。表格留给真正需要对比的数据
7. **去编号化**：能用自然过渡的别用"第X步"、"1. 2. 3."

### Step 3: 分步精修

- **结构与逻辑**：审视段落安排，优化逻辑顺序
- **句式变换**：打破单调，长短结合
- **词语润色**：替换平淡词汇
- **情感与细节**：补充感官细节、个人感受
- **去除冗余**：删掉套话、重复信息、机械表述

### Step 4: 一致性检查

确保优化后的文章忠于原意，风格统一，信息准确。

### Step 5: 通读微调

模拟目标读者通读，感受节奏与流畅度，做最后调整。

## 中文 AI 高频词清单（必须消灭）

这些词/短语出现时，大概率是 AI 写的，必须替换或删除：

- 值得注意的是、不难发现、基于以上分析、综上所述
- 核心在于、本质上、关键在于、归根结底
- 首先……其次……此外……最后……总之
- 与此同时、在此基础上、在此过程中
- 具有重要意义、发挥着重要作用、扮演着关键角色
- 不仅……而且……、既……又……（密集排比时）
- 致力于、着眼于、立足于
- 深入探讨、全面分析、系统梳理
- 高效的、强大的、卓越的、出色的（空洞形容词）
- 在当今……的时代背景下

## 句式变换技巧

| AI 常见句式 | 人类写法 |
|:---|:---|
| XXX 是一种……的技术 | XXX 说白了就是…… |
| 该方案具有以下优势 | 这个方案好在哪呢 |
| 通过……实现了…… | 搞了一通……终于…… |
| 在此过程中 | 这一步 |
| 值得注意的是 | 有意思的是 / 说个细节 |
| 综上所述 | 回过头来看 / 说到底 |
| 这充分说明了 | 这就说明一个问题 |

## Output Format

```
## 原文 AI 特征分析
[2-3 个主要问题]

## 核心优化策略
[3-5 个着手点]

## 优化亮点说明
[1-2 处关键修改示例，解释为什么这么改]

## 优化后的文章
[完整润色版本]
```

## Constraints

- 忠于原意：核心信息、关键数据不得篡改或遗漏
- 风格匹配：优化后风格需符合原文主题和目标受众
- 自然为本：避免过度修饰或炫技
- 逻辑严谨：不破坏原文逻辑结构
- 杜绝新"AI 味"：优化后的文本必须彻底摆脱机器痕迹
- 保留 markdown 格式：如果原文是 .md 文件，输出也保持 markdown
