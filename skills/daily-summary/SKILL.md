# 每日工作总结（Daily Summary）

根据当天的 git commit messages 中的「修改描述」字段内容，自动生成一份简洁的日报风格工作总结。

## 何时执行

- 用户说「帮我总结今天的工作」「日报」「今日总结」「生成工作报告」等
- 用户要求获取今天的 commit 并总结

## 执行步骤

### 1. 获取今天的 commit messages

运行以下命令获取当天所有 commit 的完整 message:

```bash
git log --since="midnight" --format="%B---SEPARATOR---"
```

如果当天没有 commit, 提示用户「今天暂无提交记录」并结束。

### 2. 提取修改描述

从每条 commit message 中提取「修改描述：」后面的内容。去掉 `[feat]` `[fix]` `[refactor]` 等前缀标签, 只保留实际的功能描述文字。

### 3. 合并与归纳

将所有修改描述进行合并归纳:

- 相近的功能点合并为一条
- 最终条目数: 最少 1 条, 最多 4 条
- 每条用简洁的中文描述, 一句话说清做了什么
- 不需要非常详细, 像日报汇报风格

### 4. 输出格式（必须严格遵循）

```
{月}.{日}
1.第一条总结内容
2.第二条总结内容
3.第三条总结内容
```

**格式要求:**

- 第一行是日期, 格式为 `月.日`, 例如 `2.6`（不补零）
- 每条总结前面是序号加英文句号, 紧跟内容, 序号和内容之间无空格
- **禁止使用 markdown 语法**（不要加粗、不要标题、不要列表符号）
- **禁止使用括号**（不要圆括号、不要方括号、不要书名号）
- **禁止使用 emoji 或其他特殊符号**
- 纯文本输出, 可以直接复制粘贴到聊天工具发送
- 语言风格: 简洁平实, 像写给领导看的日报

## 示例

输入 commit messages:

```
修改描述： [feat] 帧头导出支持 MP4 视频并修复导出 0 帧问题；导出服务 JSON 改为使用 orjson。
修改描述： [fix] 精简视频裁剪页面：移除分析AEB报警等复选框，优化布局
修改描述： [feat] 回灌页面设备IP默认改为0.0.0.0，选网卡自动填充
修改描述： [fix] 修复视频播放窗口状态切换的AttributeError
```

输出:

```
2.6
1.实现帧头导出支持MP4视频,修复导出0帧问题,导出JSON改用orjson提升性能
2.精简视频裁剪页面,移除多余复选框并优化布局
3.回灌页面设备IP默认改为0.0.0.0,选网卡后自动填充IP
4.修复视频播放窗口状态切换报错
```

## 注意事项

- 只读取当天的 commit, 不读取昨天或更早的
- 如果 commit 很多且内容相近, 务必合并归纳, 不要超过 4 条
- 不要照搬 commit message 原文, 要用自然语言重新组织
- 输出内容中不要出现英文技术术语的缩写, 尽量用中文表达
- 最终结果直接输出纯文本, 不要包裹在代码块中
